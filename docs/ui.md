以下是从专业 UI/UX 设计师视角梳理的界面问题，按优先级分类。

   ────────────────────────────────────────────────────────────────────────────────

   P0｜第一眼就能看到的专业感硬伤

   1. 右下角 TRAE SOLO Badge 遮挡核心内容

   • 截图位置：所有截图右下角
   • 问题：Badge 固定在画布上方，覆盖了右侧配置面板底部、状态栏和部分图表区域。浅色模式下还遮挡了“字体大小”等展开项。对科研工具而言，这像广告水印，严重削弱专业感。
   • 建议：彻底移除或收进「关于」菜单；若必须保留，放到窗口标题栏或启动页，不可遮挡工作区。

   2. 深色主题图表画布与 UI 背景脱节

   • 截图：plot3d_home.png、plot3d_state_3d_surface.png
   • 问题：深色模式下，中央 Plotly 画布仍是浅灰蓝背景，与周围深蓝/黑色面板形成强烈反差，不像一个统一的设计系统。
   • 建议：深色模式应将 Plotly 画布背景设为透明或深色（如 #1a1a2e / #0f172a），网格线、坐标轴文字同步调亮。

   3. 浅色主题对比度与层级崩塌

   • 截图：plot3d_state_light.png、plot3d_state_panel_expanded.png
   • 问题：
       • 输入框背景与面板背景几乎同色，缺乏边框，用户难以识别可编辑区域。
       • 数据表格选中行使用高饱和青绿，与整体冷灰蓝调冲突。
       • 右侧配置面板标题“图表配置”与背景融合，层级感弱。
   • 建议：输入框统一加 1px 边框；选中行改用主题主色的低饱和版本；面板标题加粗并拉开间距。

   ────────────────────────────────────────────────────────────────────────────────

   P1｜布局与信息架构问题

   4. 三栏布局比例失衡

   • 截图：所有截图
   • 问题：左侧数据表格 + 图层面板约 220px，右侧配置面板约 280px，中间画布被挤压。数据表格列名被截断、图层面板文本换行。
   • 建议：左侧合并为可调整宽度的侧边栏；右侧配置面板改为 280–320px 并支持垂直滚动；中间画布优先保证最小 700px 可视宽度。

   5. 图层面板高度受限

   • 截图：plot3d_state_tab_图.png（图层已叠到 2 层，面板接近撑满）
   • 问题：max-h-48（192px）对多图层管理太局促，且与数据表格挤在同一侧。
   • 建议：图层面板独立成可折叠/可调整高度的面板，或移到右侧配置区顶部，避免与数据区争夺垂直空间。

   6. 右侧面板展开后无滚动，内容被截断

   • 截图：plot3d_state_panel_expanded.png
   • 问题：展开“导出设置”“字体大小”后，最后一项被 Badge 遮挡，面板本身也没有明显滚动条提示。
   • 建议：右侧配置区加 overflow-y-auto 和自定义滚动条；底部预留安全区，避免被任何浮动元素遮挡。

   7. 状态栏信息密度过低

   • 截图：所有截图底部
   • 问题：仅显示“折线图 / 正弦函数 / 51 行 / 2 图层 / 内存 22 MB”，缺少鼠标坐标、选中数据范围、缩放比例、导出状态等科研用户需要的反馈。
   • 建议：状态栏左中右分区，左侧放数据/图层摘要，中间放坐标/缩放，右侧放主题/语言/内存。

   ────────────────────────────────────────────────────────────────────────────────

   P1｜Ribbon 工具栏体验问题

   8. 颜色映射条尺寸过小

   • 截图：plot3d_home.png Ribbon 右侧
   • 问题：色条高度仅约 20px，相邻色图差异难辨，且没有当前选中态的明确边框。
   • 建议：色条高度 ≥ 28px，加当前选中边框，hover 显示色图名称 tooltip。

   9. “拟合” Ribbon 出现大量未实现/不可点的功能按钮

   • 截图：plot3d_state_tab_拟合.png
   • 问题：Ribbon 上排列了“对数、幂律、高斯、S 曲线、多峰”等按钮，但若底层仅实现线性/多项式/指数，这些按钮会让用户误以为功能齐全，属于界面与能力 mismatch。
   • 建议：未实现的功能按钮应 disabled 并带 tooltip 说明“即将推出”，或干脆不显示。

   10. Ribbon 分组标签可视层级弱

   • 截图：plot3d_home.png
   • 问题：“图表类型”“标题”“颜色映射”等分组 label 字号小、颜色淡，用户难以快速扫描。
   • 建议：分组 label 使用小写大写字母或加粗，并增加与按钮之间的间距。

   ────────────────────────────────────────────────────────────────────────────────

   P2｜图表与组件细节

   11. 默认标题使用占位符“图表标题”

   • 截图：所有截图中央顶部
   • 问题：占位符直接渲染在图表上，显得半成品。导出后如果用户忘记修改，会得到一张带“图表标题”的图。
   • 建议：默认不显示标题；输入框 placeholder 用灰色斜体提示“输入图表标题”。

   12. 图例重复

   • 截图：plot3d_state_light_2d.png 顶部图例
   • 问题：显示两条“正弦函数 - Y”，用户无法区分。
   • 建议：图层名称唯一化，或在图例中追加索引/数据集信息。

   13. 3D 图表在 2D 数据下渲染异常

   • 截图：plot3d_state_3d_surface.png、plot3d_state_light.png
   • 问题：选中“3D 曲面”后，由于数据只有 X、Y 列，实际渲染成 3D 散点/折线，不是曲面。右侧面板也未出现 Z 轴配置。
   • 建议：切换 3D 图表类型时，若数据不满足维度要求，应弹出提示或自动禁用该选项，而不是给出错误预览。

   14. 右键菜单选项过少、无图标

   • 截图：plot3d_state_contextmenu.png
   • 问题：右键图表只有 4 个导出/重置选项，且均为文字，缺少图标辅助识别。
   • 建议：增加“复制数据”“添加标注”“快速拟合”“保存为模板”等上下文相关操作；为菜单项配 16px 图标。

   15. 配置面板标签文字竖排阅读不自然

   • 截图：右侧“标签”“单位”等字段
   • 问题：中文“标签”两字竖排显示，占用高度大且阅读效率低。
   • 建议：改为水平标签或图标 + 简短标签，节省垂直空间。

   16. 数据表格列宽与滚动体验

   • 截图：左侧数据表格
   • 问题：列宽被固定得很窄，数字被截断；表头“X 轴”标签占用空间过大。
   • 建议：列宽根据内容自适应；表头简化为列名 + 小图标，必要时 tooltip 显示完整角色。

   ────────────────────────────────────────────────────────────────────────────────

   总结

   当前界面最大的三个设计风险是：

   1. TRAE SOLO Badge 与专业定位冲突（必须移除或隐藏）
   2. 深浅色主题下画布与 UI 不协调、浅色模式对比度不足
   3. 三栏布局挤压了核心画布，右侧面板缺乏滚动与留白
